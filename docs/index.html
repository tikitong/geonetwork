#Page linked to the tikitong/geonetwork repository

This feature is intended to link point pattern recognition by deep learning with network creation. It requires the installation of specific libraries with pip install -r requirements.txt

From geospatial data (Points or Polygons) a network with nodes and edges is created based on the search for the k\footnotesize kk nearest neighbours.

You can visited the network.py file, change your geojson file name in the make_network function and run python network.py

Two files are generated:

  • nodes.geojson
  • edges.geojson

Each node has the following metadata:

  1. nodeID its ID number in the network.
  2. label its name.
  3. degree its number of connections.
  4. neighbors the ID of its connected neighbours and their labels.
  5. closeness a value reflecting its centralization in the network.

Each edge has the following metadata:

  1. node_start the ID number of the starting node.
  2. node_end the ID number of its ending node.
  3. mm_len the distance between the two nodes in km\footnotesize kmkm.

The code is generalised, we can choose the number of \footnotesize k nearest neighbours, obtain a network by label or a simply a linked network. The visualization of the network can also be done on kepler.gl or any other GIS.

Here is an example of a generated map with a public dataset for the search of the two nearest neighbors.

The dataset comes from the google community forum and is provided by a user called KenGrok. It contains 2679 unsorted hand-collected archaeological structures in the Arabian Peninsula and eastern Africa. Such as kite, gate or bullseye.

Please click on a node and an edge to see the information displayed.

This is interesting because we can now build and visualise the archaeological geometry of the country.

Thanks to this network we can establish correlations between the localized structures such as, in our case:

  • if we have an artefact of type bullseye there is a 70% chance that its nearest neighbour is a triangle .
  • if you have an artefact of type gate there is a 96% chance that in a window of 10km2\footnotesize km^2km2 you will find at least one gate .

These correlations are fictitious and depend on the data in the file used to generate the network.

Now imagine having these kinds of statistics for an entire territory like Saudi Arabia.
